Diversified Neural Networks: Defeating Adversarial Attacks via Numerous Orthogonal Variants

DivNet brings the classic security principle of software diversification to deep neural networks. Instead of deploying one identical model everywhere, it enables many distinct variants so that an adversarial example that succeeds on one instance is unlikely to succeed on another.

Ying Meng and Qiang Zeng

George Mason University, Fairfax, VA, USA

The 24th Annual International Conference on Mobile Systems, Applications and Services (MobiSys ’26), June 21–25, 2026, Cambridge, United Kingdom

Paper Code Slides

The Problem

DivNet starts from a practical deployment issue that is easy to miss in standard defense evaluations. In many real systems, the same neural network is deployed identically across devices or services. Under this uniform deployment model, one successful adversarial example can often be reused to compromise every instance that shares the same defense.

Existing defenses often focus on craft-time AE success: how hard it is for an attacker to generate a successful adversarial example. DivNet emphasizes attack-time AE success: whether that crafted example actually succeeds against the victim system during deployment.
  • A low craft-time success rate only means the attacker may need more attempts to find an adversarial example.
  • A low attack-time success rate means the crafted example does not reliably transfer to deployed victims, which is the more practical security goal.
  • This distinction is central to DivNet: the system is designed to break attack reuse across identical deployments.

Method Overview

The key insight in the paper is that, in a large neural network, some weights are comparatively unimportant for benign inputs but critical for adversarial examples. DivNet performs a one-time offline analysis to find those weights, ranks them by asymmetric significance, and identifies safe modification ranges that preserve benign recognition quality while changing adversarial behavior.

  • Weight ranking: DivNet analyzes which weights matter differently to benign inputs and adversarial examples.
  • Selective modification: It modifies top-ranked weights within validated ranges instead of pruning blindly or changing weights at random.
  • Variant generation: Different subsets of modified weights produce many model variants that behave similarly on benign inputs but diverge on adversarial inputs.
  • Deployment model: Because variants are generated offline and then deployed as ordinary ASR models, the defense adds zero runtime overhead.

In short, DivNet does not rely on runtime filtering, ensembles that must execute in parallel, or generic randomness. It changes the deployment assumption itself: instead of a single homogeneous model population, it creates a large family of orthogonal variants.

Key Results

Across the paper and the 20-second pitch slides, DivNet is positioned as a practical defense with strong deployment-time benefits. The reported results show that it can generate an enormous number of distinct variants while preserving utility and resisting a broad range of attacks, including adaptive ones.

Astronomically many orthogonal variants Independent generation produces naturally distinct model instances rather than a small fixed set of alternatives.
Preserved benign accuracy The paper reports minimal degradation on benign inputs, which is necessary for practical deployment.
Defeated adversarial inputs Attack transfer drops because success on one variant no longer implies success on another.
Withstood adaptive attacks Evaluation covers unseen attacks and adaptive strategies rather than only the attacks used during offline diversification.
Zero runtime overhead The defense cost is paid offline during analysis and variant generation, not during inference.
Deployment-oriented security Its main contribution is reducing attack-time AE success in realistic deployments, not merely increasing attacker effort at craft time.
